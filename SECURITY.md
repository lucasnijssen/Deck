# Security Policy

## Reporting a Vulnerability

Please do not file public GitHub issues for security-sensitive problems.

Instead:

- Use GitHub Security Advisories if the repository has them enabled
- Or contact the maintainers privately with reproduction details, impact, and affected versions

## What to Include

- A clear description of the issue
- Steps to reproduce
- Expected impact
- Any logs, screenshots, or proof-of-concept material needed to understand the issue

We will aim to acknowledge reports quickly and coordinate a fix before public disclosure when possible.
